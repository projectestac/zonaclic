Importing activities from Clic 3.0 to JClic

JClic, apart from allowing you to create your own projects with the extension .jclic.zip, allows you to inport the packets of activities from Clic 3.0, which may have the extensions .PAC or .PCC.

This is a useful possibility if you have previously created activities using Clic 3.0 and you want to adapt them to JClic.

The importing of activities is done from the menu Open in JClic author. You must indicate the type of file PAC or PCC in the window Open file, and look for and select the packet you want, pressing Open.

JClic author will place the files used in the media library and the activities in the activity tab; from there they can be modified like any other activity created with JClic.

In the tab Activities, these appear under the name and an extension (ASS, PUZZ, TXA...) which activities created with JClic do not have.This extension is not obligatory, and can be changed or eliminated by modifying the name of the activitiy.

Once they have been imported, the packets of activities from Clic 3.0 are automatically changed into JClic projects.The only thing to be done is to save the opened project as a jclic.zip file using JClic author, which is actually the only option that the programme offers. The place where the packets are to be saved must be indicated . This is predefined, being the same folder where the file PAC o PCC is. A good place to put these projects is C:\Programme files\JClic\projects

After going through this procedure, the old packet will be changed to a project and can be seen from JClic.
