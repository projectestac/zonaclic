Proceso de carga de los applets

El JClic se ha creado con una herramienta de programación llamada "Java", que hace posible que una misma aplicación pueda funcionar en diversos tipos de ordenadores, sistemas operativos y navegadores. Las aplicaciones Java que se encuentran incrustadas en una página web se llaman "applets".

Por razones de seguridad, nuestros applets traen una firma digital que garantiza que han sido creadas por el equipo de desarrollo del JClic y que no realizan acciones contrarias a la privacidad o integridad de los datos de vuestro ordenador.

Cuando se entra por primera vez en una página que contiene un applet JClic se ejecuta un módulo javascript que comprueba si el ordenador tiene instalado el motor Java. En caso negativo aparece un mensaje que redirige la navegación hacia la página de descarga del sistema Java.

La puesta en marcha del motor Java puede tardar unos minutos, durante los cuales se visualiza un rectángulo gris en la zona donde debería aparecer el applet. En sistemas Windows, cuando se haya cargado aparecerá un icono parecido a éste en la barra de tareas:

Icono con Java 1.3.1 Icono con Java 1.4 y 1.5

Este icono seguirá mostrándose mientras siga abierta alguna ventana del navegador. Mientras esté visible, el motor Java estará en memoria y la carga de los applets se realizará a mayor velocidad.

A continuación aparecerá una ventana de diálogo que os pide permiso para ejecutar el applet:

Al hacer clic en Grant always (o en Conceder siempre si aparece en español) los applets JClic se pondrán en marcha automáticamente sin volver a pedir permisos.

Cuando hayáis validado esta pantalla el motor Java descargará los ficheros necesarios para poner en marcha JClic. Este proceso puede tardar también unos cuantos minutos, dependiendo de la velocidad de la conexión a Internet, pero sólo se realizará la primera vez que visitáis un applet JClic, o cuando haya alguna nueva versión disponible en el servidor.

El último paso consiste en la carga del paquete de actividades que estéis visitando. Eso puede tardar más o menos dependiendo del tamaño de este paquete. Hay paquetes que se cargan enteros al inicio, y otros que sólo descargan los componentes básicos y van solicitando al servidor los ingredientes (imágenes, sonidos...) a medida que se van necesitando. En este segundo caso el paquete arranca más deprisa, pero pueden producirse retrasos en el paso de una actividad a otra.

Una animación en la barra de tareas de JClic indica que se está descargando algún fichero y hay que esperar:

Cuando el applet se haya puesto en marcha podéis pasar de una actividad a la otra utilizando los botones internos de navegación de JClic.
