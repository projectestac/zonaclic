Installation of Clic 3.0 in a network

We recommend printing and reading attentively this document before 3.0 carrying out the installation of Click in a classroom connected in net|network

Click 3.0 incorporates a new system of reports that a database Access 1.1 uses. In the classrooms where the computers are connected in net|network it is recommendable that all the stations share only one database, instead of having each his|hers|theirs in its|his|her|their hard disk. This centralization of the information presents the following advantages:

  • The pupils|students can use any computer of the classroom in its|his|her|their working sessions with Click activities: the same list of groups will always appear for them and users and the data of its|his|her|their work will go to a common place. When the pupils|students consult its|his|her|their working reports they will find the data accumulated of all the sessions carried out, independently of the computer that they had used in each of them.
  • The teaching staff can carry out the maintenance of the base and consult the progress of every pupil|student from only one computer, without having to move from table in table to see that he has done every group.
  • The task of making safety copies of the information, since all the data are in only one file located in the servant|server of the net|network, simplifies.

The program ClicDB allows the teaching staff to carry out the maintenance of this database: discharges and casualties|decreases of users, consultations|consultings|practices... etc.

The pupils|students can access indirectly one of the screens of ClicDB (from the Reports menu of the current user of Click) but in a modality from which it is not possible to carry out modifications in the base. To prevent the student body accessing the functions of administration it is recommended that the icon of ClicDB be shown only in the servant|server of the net|network.

During the process of installation three questions related to the implementation of the system of reports are made in net|network:

  • The installer asks that he centralizes the database in only one computer if it|he is wanted. In affirmative case the location of the shared folder of the servant|server has to be indicated to him|her, as he explains himself further on|later on.
  • He|She|It also asks whether the system of reports must be activated automatically every time that Click is started off. In one classroom the most logical thing would be to say it|him that YES. This decision can always be changed entering the menu of global options in every computer.
  • The last question refers to the icon of the program ClicDB. As it has been said, this icon allows activating the program that is used for carrying out the maintenance and every type of consultations|consultings|practices in the database. It is recommended answering negatively to this question except when Click in the servant|server is installed, since the use of ClicDB should be reserved exclusively to the teaching staff.

In this document we will do some assumptions related to the configuration of the local network:

  • We suppose that the servant|server has a folder named C:\COMUN
  • We suppose that this folder is shared with laws|rights of reading and writing for all the stations.
  • We suppose that the stations have the letter T: assigned to the folder C:\COMUN of the servant|server.
  • We suppose that in all the computers Click will be installed in a folder called C:\CLIC

Maybe its|his|her|their net|network is not configured exactly like this: perhaps it|he|she wants to install Click in another folder (for example, C:\ARCHIV~1\CLIC), that the shared folder is not called COMUN nor it|he|she is|finds in the unit|unity C:, or that the stations see the folder with a different letter to the T:. If it|he|she is like this it|he|she will have to change the appointments at units|unity and folders that are made in this document for those that correspond to its|his|her|their net|network.

These are the crossings|steps recommended to install correctly the system of reports of Click in net|network:

  1. He|She|It thinks a folder in an area of the servant|server where all the stations have laws|rights of reading and writing. For example, if in the servant|server there is it|he|she a shared folder named C:\COMUN should create in its|his|her|their interior another folder called C:\COMUN\INFORMES. Normally the stations are connected to the shared resources through a letter: let's suppose that in this case it|he|she is the T:. If it|he|she is like this, the stations should see this folder as T:\INFORMES
  2. It|He|She checks|proves that all the stations can see this folder with the normal syntax of MSEC-TWO. Click is an application of 16 bits and it|he does not accept expressions of the type \\SERVIDOR\COMUN\INFORMES. Continuing with the previous example, it should be guaranteed that all the stations are capable of seeing the shared folder with the syntax T:\INFORMES. If they can probably not find it|her it|she will have to create a link with letter to the shared resource of net|network. In W95/98/NT this can be made through the function Conectar at unit|unity of net|network of the menu Herramientas of the Explorer of Windows.
  3. Install click in the servant|server. During the process of installation it|she will have to answer:
    • That it|he|she wants to install Click in a local folder (for example, C:\CLIC)
    • That IF that it|he|she wants to use the system of reports in net|network
    • That the database must be placed in the folder C:\COMUN\INFORMES
    • That IF that it|he|she wants the system of reports to be activated automatically
    • That IF that it|he|she wants to create the icon of the program ClicDB
    Other questions are made, but they do not have relation|relationship with the system of reports: installing or not the examples, to install the utility ClicPac...
  4. When it|he|she finishes the installation in the servant|server starts off the program ClicDB. Now it|he|she has three possibilities:
    1. The groups and pupils|students discharging from hospital manually all.
    2. Registering only the work groups and to leave that the pupils|students are registered they when they start off Click for the first time.
    3. Not registering anything and giving to any pupil|student to create new groups and to users absolute freedom.
    The option 2 is recommended. Decide what you create more appropriate and, when it|he|she finishes, close the program ClicDB.
  5. We pass to the working stations: He|She|It should install Click in each of them answering:
    • That he|she|it wants to install the program in the folder C:\CLIC (or in any other local folder)
    • That IF that it|he|she wants to use the system of reports in net|network
    • That the folder of the database is T:\INFORMES
    • That IF that it|he|she wants the system of reports to be activated automatically
    • That he|she|it does not want to create the icon of the program ClicDB
  6. When it|he|she finishes the installation, if in the crossing|step 4 a different option to the b has been chosen Click in all of them will have to start off the computers (including the servant|server) and it|he will have to carry out these adjustments:
    • Activating the menu Opciones - Global Options.
    • Marking or to distance the boxes|cells Permitir to the pupils|students to create new users and Create new groups according to the taken decision.
    • Confirming everything and, when Click asks it, telling him|her that it|he|she keeps the changes to be used in the future.
  7. Check|Prove the functioning|operation of the system of reports from each of the stations.

As in any program that shares data in net|network, it is necessary that the servant|server always starts before the stations. If the servant|server is dull or is changed an error message will be shown by the allocation|assignment of the units|unity of net|network when starting off Click.

The system of reports in net|network can make Click suffer some delay|lag in passing another of an activity, especially if there are many users using the program simultaneously. With fast computers it will practically not be noticed.

It is convenient to use regularly the program ClicDB to compact the database and to clean the information that is not needed any longer. Periodically making safety copies of the file CLICDB.MDB of the servant|server is also recommended.

It is recommended that the maintenance of the database through the utility ClicDB be carried out when there are not other users using the Click program.

For more information consult the help of the Click programs and ClicDB.
