Fill-in blanks
   

In this modality you select words, letters or sentences from a text, which you hide or conceal and the user has to fill them in. Resolving each of the hidden elements can be done in several ways:

  • by writing in the blank space which can be filled in with previously chosen characters.
  • by correcting an initial expression which has mistakes.
  • by choosing possible answers from a dropdown menu.

The unknown elements may have an emerging help window that can contain a message, an image, a cartoon... This window can be activated at different times:

  • automatically when the text activity starts
  • when the written answer has a mistake
  • when the user asks for help by pressing F1.

You can also give non-visual help such as sounds.

With all these different aids you can arrange for a delay or a time limit for their appearance.

The evaluation of the fill-in blanks activities can be immediate or delayed, and in both cases you can make JClic try to detect which letters or words cause the error in an answer and make it stand out in a different colour. However, you can simply say if the answer is right or wrong without giving any clues as to where the mistake is.

In this type of activity it is best to press the Return key after having written the answer in each of the spaces. In this way JClic will know that it is time to evaluate it.
