Procés de càrrega dels applets

El JClic s'ha creat amb una eina de programació anomenada "Java", que permet a les aplicacions puguin funcionar en diversos tipus d'ordinadors, sistemes operatius i navegadors. Les aplicacions Java que es troben dins d'una pàgina web s'anomenen "applets".

Per raons de seguretat, els nostres applets porten una signatura digital que garanteix que han estat creades per l'equip de desenvolupament del JClic i no realitzen accions contràries a la privacitat o integritat de les dades del vostre ordinador.

Quan s'entra per primera vegada en una pàgina que conté un applet JClic s'executa un mòdul javascript que comprova si l'ordinador té instal·lat el motor Java. En cas negatiu apareix un missatge que redirigeix la navegació cap a la pàgina de descàrrega del sistema Java.

La posada en marxa del motor Java pot trigar uns minuts, durant els quals es visualitza un rectangle gris en la zona on hauria d'aparèixer l'applet. En sistemes Windows, quan s'hagi carregat apareixerà una icona semblant a aquesta a la barra de tasques:

Icona amb Java 1.3.1 Icona amb Java 1.4 i 1.5

Aquesta icona seguirà mostrant-se mentre tingueu oberta alguna finestra del navegador. Mentre estigui visible, el motor Java estarà en memòria i la càrrega dels applets es realitzarà més de pressa.

A continuació apareixerà una finestra de diàleg que us demana permís per a executar l'applet:

Si feu clic a Grant always (o a Conceder siempre si apareix en castellà) els applets JClic s'engegaran automàticament sense tornar a demanar permisos.

Quan hagueu validat aquesta pantalla el motor Java descarregarà els fitxers necessaris per a posar en marxa el JClic. Aquest procés pot trigar també uns quants minuts, depenent de la velocitat de la connexió a Internet, però només es realitzarà la primera vegada que visiteu un applet JClic, o quan hi hagi alguna nova versió disponible al servidor.

El darrer pas consisteix en la càrrega del paquet d'activitats que estigueu visitant. Això pot trigar més o menys depenent de la mida d'aquest paquet. Hi ha paquets que es carreguen sencers a l'inici, i altres que només descarreguen els components bàsics i van sol·licitant al servidor els ingredients (imatges, sons...) a mida que els van necessitant. En aquest segon cas el paquet engega més de pressa, però poden produir-se retards en el pas d'una activitat a una altra.
